# 升级说明模板(每次发版复制填写)

## 版本
- 自 vX.Y.Z 升级至 vX.Y.Z+1;发布日期:____;责任人:____

## 升级前检查
- [ ] 当日备份存在且可恢复(抽验)
- [ ] `selfcheck_prod` 全绿
- [ ] 阅读下方"行为变化"并确认现场影响

## 行为变化(必填,无则写"无")
> 任何默认值变更、新增审计事件、接口契约变化、时序变化都必须列明。
- 变化 1:____(影响面/是否需要操作)
- 变化 2:____

## 数据库变更
- 新增迁移版本:____(expand 型,启动自动应用;回退观察期内不删旧列)

## 升级步骤
1. 停外部入口(nginx 维护页)→ 2. 替换制品 → 3. 滚动重启 →
4. `scripts/smoke.py` 全绿 → 5. 放开入口。

## 回退方案
- 制品回退到上一版本;数据库不回滚(expand 型向后兼容);
  如涉不兼容变更,升级说明必须给出专项回退步骤。
